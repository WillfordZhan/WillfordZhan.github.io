Termius + Tailscale + tmux

目标：用 iPhone 上的 Termius 随时随地连回 Mac，通过 Tailscale 组网保证安全稳定，用 tmux 保持会话不断线（包含常驻运行 Codex）。

为什么要这样搭

  • Termius：手机端 SSH 客户端，操作方便。
  • Tailscale：不用端口映射，不怕公网 IP 变化；同一个 Tailnet 里设备互通。
  • tmux：SSH 断线/切后台也不影响任务继续跑，随时重新接回。

Mac 端：开启 SSH

  1. 系统设置 → 通用 → 共享 → 远程登录(Remote Login) → 打开。
  2. 确认你用的是自己的账号登录（例如 willfordzhan）。

iPhone（Termius）：生成 SSH Key 并授权

  1. Termius → Keychain → + → Generate Key → Type 选 Ed25519。
  2. 进入该 Key，复制 Public Key（形如 ssh-ed25519 AAAA...）。
  3. 在 Mac 上把公钥追加到：~/.ssh/authorized_keys（一行一个 key）。
nano ~/.ssh/authorized_keys
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

Tailscale：同一账号组网

  1. Mac 安装并登录 Tailscale（建议用官方 Tailscale.app）。
  2. iPhone 安装并登录 Tailscale，开启 VPN，确保手机也显示在线。
  3. 在 Tailnet 里找到 Mac 的 Tailscale IP（通常是 100.x.y.z）或 MagicDNS 名称。

Termius：添加 SSH Host

  • Host：填 Mac 的 100.x.y.z 或 MagicDNS。
  • Port：22
  • Username：你的 macOS 用户名（例如 willfordzhan）
  • Authentication：选择刚才生成的 Key

tmux：保持会话不掉线

brew install tmux

# 新建或接入名为 codex 的会话
# （已在本机准备了 codex-mobile 脚本时，直接运行 codex-mobile 即可）
tmux new -As codex

常用快捷键：

  • 断开但保留会话：Ctrl-b 然后按 d
  • 列出会话：tmux ls
  • 重新接入：tmux attach -t codex

在 tmux 里跑 Codex

建议始终在 tmux 会话里启动 Codex，这样手机切后台/断网也不会中断。

codex

常见坑：SSH 访问 Desktop 报 Operation not permitted

这是 macOS 的隐私保护（TCC）。如果你希望 SSH 会话也能读写 ~/Desktop：

  1. 系统设置 → 隐私与安全性 → 完全磁盘访问权限(Full Disk Access)
  2. 添加：/usr/sbin/sshd
  3. 断开 Termius 连接，重新 SSH 登录后再试。

更稳的替代方案：把工作目录放到 ~/Work（不放 Desktop）避免触发权限限制。

安全建议

  • 只用 SSH Key 登录，尽量关闭密码登录。
  • 尽量只在 Tailscale 内网 IP 上使用 SSH，不要对公网暴露 22 端口。
  • 丢手机时，第一时间在 GitHub/Termius/Tailscale 撤销 Key/设备。
